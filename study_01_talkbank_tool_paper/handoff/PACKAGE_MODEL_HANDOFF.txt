Purpose
This file is the package-facing handoff for turning the trained research model into a Python package that annotates .cha files. It records the current inference contract from this repo so the package can reproduce the trained model behavior instead of guessing.

- Unit of prediction: one CHAT utterance line at a time
- Expected output: the original utterance line with inline CHAT morphosyntactic error annotations inserted

Recommended packaging choice
Use a merged standalone model repo for packaging if possible.

Why
- It gives the package a single model repo to load.
- It avoids package-time PEFT adapter assembly.
- It avoids package-time embedding resize logic in the normal case.
- It reduces opportunities for tokenizer/model mismatch.

Recommended Hugging Face layout
Option A: preferred
- One private merged-model repo containing the final full model weights plus tokenizer
- The package loads only that repo with transformers

Option B: acceptable but less clean
- One private adapter repo
- The package also needs the exact base model id
- The package also needs the exact tokenizer state or the exact chat_tokens.json
- The package must add CHAT tokens, resize embeddings if tokens were added, then load the adapter

Tokenizer and embedding rule
This project uses tokenizer augmentation for CHAT-specific units and structured label pieces.

Current repo behavior
- Load the base model tokenizer
- Load the base model
- Read experiments/recon_full_comp_preserve/chat_tokens.json
- Add the CHAT tokens to the tokenizer
- If new tokens were actually added, resize token embeddings
- Then load the LoRA adapter

Practical rule
- If deploying base model + adapter: yes, resize embeddings when and only when token addition actually increases tokenizer size
- If deploying a merged full model saved with the correct tokenizer: no runtime resize should be needed
- If the tokenizer already contains the 53 CHAT tokens and the model already matches that vocab size, do not resize again

Answer to "does the embedding layer need to be resized every time?"
- No, not every time in the abstract
- Yes for the current adapter-based loading path if starting from the base model tokenizer and then adding the CHAT tokens
- No for a properly saved merged full model repo with the final tokenizer

How to merge the adapter into a standalone model
Use PEFT merge_and_unload() after reproducing the exact tokenizer state used for inference.

Safe merge order
1. Load the exact base model used in training.
2. Load the tokenizer.
3. Add the CHAT tokens from experiments/recon_full_comp_preserve/chat_tokens.json.
4. If tokens were added, resize the model embeddings.
5. Load the adapter from the private HF repo.
6. Run merge_and_unload().
7. Save the merged model and tokenizer.
8. Push the merged result to a new private HF repo.

Minimal merge example
```python
import json
from pathlib import Path

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

base_model_id = "unsloth/Meta-Llama-3.1-8B-Instruct-bnb-4bit"
adapter_repo_id = "mash-mash/talkbank-morphosyntax-annotator-final-recon_full_comp_preserve_final_seed3407"
chat_tokens_path = Path("experiments/recon_full_comp_preserve/chat_tokens.json")
out_dir = "merged-talkbank-chat-annotator"

tokenizer = AutoTokenizer.from_pretrained(base_model_id, use_fast=True)

model = AutoModelForCausalLM.from_pretrained(
    base_model_id,
    torch_dtype=torch.bfloat16 if torch.cuda.is_available() else torch.float32,
    device_map="auto",
)

chat_tokens = json.loads(chat_tokens_path.read_text(encoding="utf-8"))
chat_tokens = list(dict.fromkeys(chat_tokens))
added = tokenizer.add_tokens(chat_tokens, special_tokens=False)
if added > 0:
    model.resize_token_embeddings(len(tokenizer))

model = PeftModel.from_pretrained(model, adapter_repo_id)
model = model.merge_and_unload()

model.save_pretrained(out_dir)
tokenizer.save_pretrained(out_dir)
```
